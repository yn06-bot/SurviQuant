# Operations & CI/CD Pipeline

본 문서는 대시보드의 실시간성을 보장하고 무거운 연산 부하를 프론트엔드와 분리하기 위한 파이프라인 아키텍처 명세입니다.

## 1. 일일 데이터 갱신 및 추론 파이프라인 (Daily Update)

정적 파일 기반 대시보드에 최신 지표를 실시간으로 제공하는 역할을 수행합니다.

1. 데이터 수집 로직: 매일 장 마감 후(UTC 자정) GitHub Actions가 `yfinance`를 통해 당일 S&P 500 종목의 OHLCV 데이터를 수집합니다.

2. 병합 및 추론 로직: 수집된 최신 데이터를 기존 누적 시계열에 병합한 뒤, 사전에 학습되어 저장된 `model.pkl` 파일에 통과시켜 내일 자 AI Score를 추론합니다.

3. 배포 로직: 추론된 점수를 프론트엔드가 즉시 읽을 수 있도록 `scores.json` 정적 파일로 덮어쓰기 하여 GitHub에 커밋합니다. 결측치가 5% 이상 발생 시 즉시 Alert를 발생시킵니다.

## 2. 주간 모델 재학습 파이프라인 (Weekly Retrain)

새로운 시장 패턴을 반영하고 Data Drift로 인한 C-index 성능 하락을 방지합니다.

1. 재학습 로직: 매주 일요일 자정, 최근 1주일간의 신규 레이블링(수익/손실 10% 도달 여부 확정 건)을 훈련 셋에 반영하여 Random Survival Forest 파라미터를 갱신합니다.

2. 롤백 정책: 재학습된 모델의 검증 셋 성능(RMSE 통제 및 C-index 검증)이 기존 운영 모델 대비 5% 이상 하락할 경우, 과적합으로 간주하여 신규 `model.pkl` 덮어쓰기를 취소하고 이전 버전을 유지합니다.
